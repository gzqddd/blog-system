极简博客项目运行说明文档
项目简介
本项目是一款支持多类型内容发布的极简博客系统，支持文章、图片集、音乐、视频等形式的博客发布，前端采用 React 实现炫酷的视觉效果，后端基于 Node.js + Express 构建，数据库使用 MongoDB 存储数据。

一、前置环境准备
1.1 Node.js 下载与安装配置
1.1.1 下载
访问 Node.js 官方下载地址：https://nodejs.org/zh-cn/download/
推荐下载 LTS 长期支持版（如 v18.x/v20.x），根据操作系统选择对应安装包（Windows/macOS/Linux）。
1.1.2 安装（Windows）
双击下载的.msi 安装包，点击「下一步」；
勾选「I accept the terms in the License Agreement」，点击「下一步」；
选择安装路径（默认即可），点击「下一步」；
勾选「Add to PATH」（关键！自动配置环境变量），点击「下一步」；
保持默认选项，点击「下一步」→「安装」→「完成」。
1.1.3 安装（macOS）
双击下载的.pkg 安装包，按照引导完成安装；
1.1.4 验证安装
打开命令行工具（Windows：CMD/PowerShell；macOS：终端）；
执行以下命令，若显示版本号则安装成功：

    node -v  # 查看Node.js版本
    npm -v   # 查看npm版本（Node.js自带npm包管理器）

1.2 MongoDB 下载与安装配置
1.2.1 下载
访问 MongoDB 社区版下载地址：https://www.mongodb.com/try/download/community
选择对应操作系统（Windows/macOS）和版本，下载安装包；
（可选）推荐使用 MongoDB Compass（可视化工具）：下载地址https://www.mongodb.com/try/download/compass
1.2.2 安装（Windows）
双击 msi 安装包，选择「Complete」（完整安装），点击「Next」；
勾选「Install MongoDB as a Service」，设置 Service Name 为「MongoDB」，选择「Run service as Network Service user」，点击「Next」；
勾选「Install MongoDB Compass」（可选，可视化工具），点击「Next」→「Install」→「Finish」。
1.2.3 安装（macOS）
双击 pkg 安装包，按引导完成安装；
1.2.4 启动 MongoDB 服务
Windows
按下Win + R，输入services.msc，回车打开服务列表；
找到「MongoDB Server」（或命名为 MongoDB），右键「启动」；
（可选）设置开机自启：右键「属性」→「启动类型」选择「自动」。
macOS:

    # 启动MongoDB服务
    brew services start mongodb-community

    # 验证服务是否启动
    brew services list | grep mongodb

1.2.5 验证 MongoDB 连接
打开命令行，执行以下命令进入 MongoDB 交互终端：

    mongosh  # v6.0+版本

若进入test>交互提示符，说明连接成功；
（可选）打开 MongoDB Compass，使用默认连接字符串mongodb://localhost:27017，点击「Connect」，可可视化管理数据库。

二、项目运行步骤
2.1 获取项目源码
若从 GitHub 克隆：

    git clone [你的GitHub仓库地址].git
    cd [项目文件夹名称]
若为压缩包：解压源码到本地任意目录（避免中文路径）。
2.2 VSCode 打开项目
    1.打开 VSCode，点击左侧「文件」→「打开文件夹」，选择解压后的项目根目录；
    2.（推荐）安装 VSCode 必备插件：
    ES7+ React/Redux/React-Native snippets（React 代码提示）
    Prettier（代码格式化）
    ESLint（代码检查）
2.3 安装项目依赖
2.3.1 区分前后端（若项目分离）
若项目为前后端分离结构：
进入后端目录（如backend）：

    cd backend
    npm install  # 安装后端依赖

进入前端目录（如frontend）：

    cd frontend
    npm install  # 安装前端依赖

若项目为单目录（前后端同目录）：
    1.在项目根目录打开 VSCode 终端（快捷键 `Ctrl +``）；
    2.执行以下命令安装所有依赖：

    npm install

2.3.2 依赖安装说明
首次启动必须执行npm install，会根据项目根目录的package.json自动安装所有依赖包（如 React、Express、mongoose 等）；
若安装缓慢，可切换淘宝镜像：

    npm config set registry https://registry.npm.taobao.org

2.4 配置环境变量
在项目根目录（后端目录）创建.env文件，添加以下配置（根据实际需求调整）：

    # MongoDB连接字符串（本地默认）
    MONGO_URI=mongodb://localhost:27017/blog_db
    # 端口配置
    PORT=5000  # 后端服务端口
    REACT_APP_API_URL=http://localhost:5000  # 前端请求后端的地址

说明：blog_db为数据库名称，首次启动会自动创建。

2.5 启动项目
2.5.1 启动后端服务
若为分离结构，进入后端目录执行：

    cd backend
    npm start  # 或 node server.js / nodemon server.js（热重载）

若为单目录，在根目录执行：

    npm run server  # 需在package.json的scripts中配置"server": "node server.js"

启动成功提示：Server running on port 5000 / Connected to MongoDB。

2.5.2 启动前端服务
若为分离结构，进入前端目录执行：

    cd frontend
    npm start

若为单目录，在根目录执行：

    npm run client  # 需在package.json的scripts中配置"client": "react-scripts start"

启动成功后，会自动打开浏览器访问http://localhost:3000（默认前端端口），若未自动打开，手动在浏览器输入该地址即可。

2.6 首次运行验证
访问http://localhost:3000，可看到博客首页；
点击「注册」创建账号，「登录」后可发布文章 / 图片集 / 音乐 / 视频；
所有数据会存储在本地 MongoDB 的blog_db数据库中。
三、核心功能说明
用户模块：注册、登录、个人中心（头像 / 简介修改）；
博客发布：支持文章（带封面 / 摘要）、图片集（多图上传）、音乐（外链 / 本地文件）、视频（外链 / 本地 MP4）；
博客展示：首页瀑布流展示、分类筛选、详情页预览；
交互效果：玻璃拟态、霓虹边框、3D 动效、流光按钮等炫酷视觉效果。
四、常见问题排查
4.1 Node.js/npm 命令找不到
问题原因：安装时未勾选「Add to PATH」，环境变量未配置；
解决：重新安装 Node.js 并勾选 PATH，或手动配置环境变量（将 Node.js 安装目录的bin文件夹添加到系统 PATH）。
4.2 MongoDB 启动失败
Windows：检查端口 27017 是否被占用（执行netstat -ano | findstr :27017），关闭占用进程；
macOS：执行brew services restart mongodb-community重启服务；
若提示权限不足，以管理员 /root 身份启动命令行。
4.3 依赖安装失败
删除项目根目录的node_modules文件夹和package-lock.json文件，重新执行npm install；
确保网络正常，或切换 npm 镜像（淘宝镜像）。
4.4 前端启动后无法访问后端接口
检查后端服务是否启动（端口 5000 是否监听）；
检查.env文件中的REACT_APP_API_URL是否与后端端口一致；
检查后端是否配置跨域（需安装cors包并启用）：

    // 后端server.js中添加
    const cors = require('cors');
    app.use(cors());
    
五、补充说明
若需部署到服务器，需配置 MongoDB 远程连接、端口开放、环境变量等；
项目所有炫酷视觉效果基于纯 CSS 实现，无需额外安装插件；
本地开发时，MongoDB 默认无密码验证，生产环境需配置账号密码。
